# HW 05: Structural Patterns (Facade, Adapter)

Цей проєкт демонструє використання структурних патернів проєктування (Facade, Adapter) на прикладах з практичного TypeScript-контексту.

## Структура проєкту
---

* `/src/DirectoryReport.ts` – опис інтерфейсу `DirectoryReport`, який визначає єдину структуру передачі результатів аналізу файлової системи (кількість файлів, папок, загальний розмір та статистика розширень).
* `/src/DirectoryAnalyzer.ts` – реалізація класу `DirectoryAnalyzer`, що виконує рекурсивний обхід вказаної директорії та збирає метрики файлової системи.
* `/src/ReportAdapter.ts` – інтерфейс `ReportAdapter`, який задає загальний контракт (`export(report: DirectoryReport): string`) для всіх адаптерів форматування.
* `/src/JsonReportAdapter.ts` – адаптер `JsonReportAdapter`, який серіалізує звіт у формат JSON.
* `/src/CsvReportAdapter.ts` – адаптер `CsvReportAdapter`, який перетворює звіт у структуровані CSV-рядки.
* `/src/XmlReportAdapter.ts` – адаптер `XmlReportAdapter`, який конвертує звіт у XML-документ.
* `/src/AnalyzerFacade.ts` – низькорівневий фасад `AnalyzerFacade`, що інкапсулює взаємодію між аналізатором `DirectoryAnalyzer` та обраним `ReportAdapter`.
* `/src/ReportManager.ts` – високорівневий фасад `ReportManager`, що керує життєвим циклом звіту: вибір адаптера за форматом, запуск аналізу через низькорівневий фасад, перевірка існування папки `reports/`, збереження згенерованого звіту із міткою часу та обробка помилок.
* `/src/main.ts` – точка входу в CLI-застосунок.

## Використані патерни
---

* **Facade**: спрощує взаємодію користувача зі складною логікою рекурсивного аналізу та форматування звітів за допомогою ієрархії фасадів. Високорівневий фасад `ReportManager` повністю приховує деталі створення об'єктів, роботи з файловою системою та форматування від точки входу `main.ts`, тоді як `AnalyzerFacade` об'єднує низькорівневі процеси сканування та серіалізації.
* **Adapter**: забезпечує сумісність між єдиною структурою аналізу `DirectoryReport` та різними форматами експорту (`JSON`, `CSV`, `XML`). Це дозволяє легко додавати нові класи адаптерів без жодних змін у коді аналізатора або фасадів.

## Запуск та компіляція
---

### 1. Встановлення залежностей
```bash
npm install
```

### 2. Запуск у режимі розробки (через `ts-node`):
```bash
npx ts-node src/main.ts "<шлях_до_директорії>" <формат>
```
*Наприклад:*
```bash
npx ts-node src/main.ts "." xml
```

### 3. Компіляція та запуск готового JS-коду:
```bash
npm run build
node dist/main.js "<шлях_до_директорії>" <формат>
```
